## 1. 系统定位与学习目标

### 1.1 系统是什么

这不是一个「把星星拉成线条」的科幻特效演示，而是一个**以惯性参考系变换为核心**的科学可视化系统：

| 层次 | 用户应获得的理解 |
|------|------------------|
| 体验层 | 驾驶飞船加速到 0.9c、0.99c，感受星空与仪表盘变化 |
| 对比层 | 同一时刻，飞船内看到什么 vs 地球上的人看到什么 |
| 解释层 | γ 因子、固有时间、测量长度、光传播延迟的区别 |
| 反思层 | 通过任务与测验纠正「尺缩 = 视觉压扁」等误解 |

### 1.2 狭义相对论的边界（本系统不模拟）

- 广义相对论：引力、黑洞、引力透镜、引力钟慢
- 非惯性系：加速中的等效引力（可作为 Future Work 单独说明）
- 超光速：β 上限建议锁在 0.999 以下

## 2. 总体架构建议

### 2.1 推荐技术栈

```
┌─────────────────────────────────────────────────────────┐
│  表现层：Three.js 场景 + WebXR + HTML/CSS HUD           │
├─────────────────────────────────────────────────────────┤
│  物理层：纯函数 relativity.js（γ、尺缩、多普勒、光行差） │
├─────────────────────────────────────────────────────────┤
│  状态层：App state（β, frame, viewMode, earthTime…）    │
├─────────────────────────────────────────────────────────┤
│  数据层：DataLogger → JSON/CSV 导出                     │
└─────────────────────────────────────────────────────────┘
```

| 模块 | 推荐选型 | 理由 |
|------|----------|------|
| 3D 渲染 | Three.js r16x+ | 生态成熟，内置 WebXR |
| 构建 | Vite | 快速 HMR，适合 WebXR 调试 |
| VR 入口 | `VRButton` + `renderer.xr.enabled` | 浏览器原生 WebXR，免安装 |
| 驾驶舱 UI | CSS2DRenderer / Canvas 纹理 Sprite | VR 中可读性较好 |
| 物理 | 独立 `physics/` 模块 | 便于单元测试与论文复现 |
| 时空图 | Canvas 2D 离屏绘制 | 轻量，不占用 GPU 主循环 |

### 2.2 核心状态机

建议维护以下全局状态（本项目 `App.js` 已部分实现）：

```javascript
{
  beta: 0,              // v/c，建议 clamp 到 [0, 0.999]
  frame: 'earth',       // 'earth' | 'ship' | 'side-by-side'
  viewMode: 'measured', // 'measured' | 'observed'
  earthTime: 0,         // 地球参考系累计时间（年）
  earthDistance: 4.24,  // 地球系到目标距离（光年）
  paused: false,
  cameraMode: 'chase'   // 建议扩展：'cockpit' | 'earth-fixed' | 'split'
}
```

### 2.3 渲染管线（每帧）

```
输入（键盘/VR 手柄/滑块）
    ↓
更新 β、飞船位姿、earthTime
    ↓
computeRelativityState() → γ, shipTime, lengthRatio, ETA…
    ↓
按 frame + viewMode 分支更新各可视化模块
    ↓
主相机 / 副相机 / XR 双眼渲染
    ↓
HUD + 日志快照
```

## 3. 物理核心：必须算对什么

### 3.1 洛伦兹因子与基本公式

速度参数 β = v/c，γ = 1/√(1−β²)。

| 物理量 | 地球系（静止观察者） | 飞船系（与飞船共动的惯性系） |
|--------|----------------------|------------------------------|
| 飞船上时钟读数（固有时间） | Δt_earth（外界观测到的坐标时） | Δτ = Δt_earth / γ |
| 沿运动方向测量长度 | L = L₀/γ（尺缩） | L₀（静止长度，不收缩） |
| 到目标距离 | D_earth | D_ship = D_earth / γ |
| 到达 ETA | D_earth / β | (D_earth/β) / γ |

本项目 `src/physics/relativity.js` 已实现上述计算，**所有可视化模块应只调用此模块，避免散落重复公式**。

### 3.2 关键物理澄清（制作前必读）

**误区 1：飞船「头部时钟」和「尾部时钟」在飞船静止参考系中走速不同**  
→ 在**匀速直线惯性运动**中，飞船内各点固有时间速率相同。  
→ 与地球钟对比时，整艘飞船共用一个 τ，不存在「船头更慢、船尾更快」。

**误区 2：尺缩就是肉眼看到的「变扁」**  
→ 尺缩是**在同一参考系中用刚性标尺同时测量**两端得到的结果。  
→ 肉眼看到的是**光到达眼睛后的图像**，包含传播延迟、光行差、Penrose-Terrell 旋转，二者不必一致。

**误区 3：前进方向与后退方向钟慢不同**  
→ 钟慢效应本身不区分前后；**前后视觉差异**主要来自多普勒频移与相对论光行差（星星颜色、星空聚集方向）。

下文第 5、6 节将分别说明：如何在 UI 上正当呈现「时钟对比」，以及如何正确模拟尺缩。

## 4. 双视角设计：飞船内 vs 静止观察者

### 4.1 三种参考系呈现模式

| 模式 | 相机位置 | 场景如何变换 | 教学用途 |
|------|----------|--------------|----------|
| **Earth（地球/静止系）** | 固定在太阳系旁或第三人称追尾 | 飞船高速运动，外部物体长度按地球系测量收缩 | 理解「外界看到飞船变短」 |
| **Ship（飞船系）** | 驾驶舱第一人称或船体固定 | 外界星空/距离按飞船系测量收缩 | 理解「宇航员觉得路程变短」 |
| **Side-by-side（并列）** | 左右分屏或画中画 | 同一 β 同步渲染两套解释 | 最强对比，适合课堂 |

### 4.2 飞船内视角（Cockpit View）

**实现要点：**

1. **相机挂载**：将 `PerspectiveCamera` 或 XR 头显锚点设为飞船子节点，位于驾驶舱眼位（例如 `(0, 0.35, -0.3)` 船体坐标）。
2. **船体静止**：在飞船系下，驾驶舱内饰、仪表盘、扶手不应做尺缩 scale（它们是静止长度 L₀）。
3. **窗外景象**：对星空应用光行差 + 多普勒；对目标恒星距离显示 `D_ship = D_earth/γ`。
4. **仪表盘**：显示 τ（固有时间）、γ、β，以及与地球通讯延迟。

```javascript
// 伪代码：切换到驾驶舱视角
cockpitCamera.position.set(0, 0.35, -0.3);
spacecraft.group.add(cockpitCamera);
// 每帧：cockpitCamera 随飞船旋转，不再使用 chase cam lerp
```

### 4.3 静止观察者视角（Earth-Fixed / External View）

**实现要点：**

1. **相机固定**在世界坐标，例如 `(80, 40, 120)` 俯视太阳系。
2. **飞船运动**：飞船 mesh 沿航线移动，速度 = β × v_max。
3. **飞船外观尺缩**：在地球系 measured 模式下，沿速度方向 scale：  
   `spacecraft.group.scale.z = (1/γ) * baseScale`（注意轴向与模型 nose 方向对齐）。
4. **时间显示**：场景旁浮动「地球钟」与「飞船钟」双面板，地球钟走 earthTime，飞船钟走 shipTime。

### 4.4 对比呈现的最佳实践

| 方案 | 描述 | VR 适配 |
|------|------|---------|
| **A. 分屏** | 左半屏驾驶舱，右半屏外部 | 桌面优秀；VR 需左右眼各看一侧，体验一般 |
| **B. 画中画** | 主视角驾驶舱，角落嵌入外部小窗 | VR 可用曲面 UI 面板悬浮在舱内 |
| **C. 一键切换** | 按键在 cockpit ↔ external 间切换 | **VR 推荐**：沉浸感最好 |
| **D. 双摄像机 RenderTarget** | 两路渲染到纹理，贴在舱内「监视器」上 | 教学效果极佳：舱内看「地球视角直播」 |

**推荐组合**：VR 默认驾驶舱 + 舱内监视器（方案 D）；桌面默认 Side-by-side 或分屏。

### 4.5 双摄像机实现骨架

```javascript
// 第二摄像机渲染到纹理
const rt = new THREE.WebGLRenderTarget(512, 288);
const earthCam = new THREE.PerspectiveCamera(60, 16/9, 0.1, 8000);
earthCam.position.set(100, 50, 150);
earthCam.lookAt(shipPosition);

// 主循环中（XR 外或 pre-render pass）
renderer.setRenderTarget(rt);
renderer.render(scene, earthCam);
renderer.setRenderTarget(null);

// 将 rt.texture 贴到驾驶舱监视器 Mesh
monitorMaterial.map = rt.texture;
```

## 5. 钟慢效应：时钟对比的实现方案

### 5.1 应对比哪两只钟？

| 对比类型 | 时钟 A | 时钟 B | 物理含义 | 推荐展示位置 |
|----------|--------|--------|----------|--------------|
| **主对比** | 地球协调时 | 飞船固有时间 τ | 真正的钟慢效应 | HUD 双行 + 舱内主仪表 |
| **旅程对比** | 出发时刻地球钟 | 到达时地球钟 vs 飞船钟 | 双生子悖论情境 | 任务结算页 |
| **光传播对比** | 飞船钟「现在」 | 地球看到的飞船钟（延迟后） | 非同时性 | 高级模块 |

### 5.2 「前进方向 vs 反方向」时钟应如何理解？

用户常希望看到「船头方向」与「船尾方向」两只钟。在**匀速惯性系**中：

- **飞船共动系内**：前后钟读数相同（同时校准后始终保持同步）。
- **可做的正当差异**（不是钟慢，而是光学/通讯效应）：

| 效果 | 前进方向（船头窗外） | 反方向（船尾窗外） | 实现方式 |
|------|----------------------|--------------------|----------|
| 多普勒 | 蓝移（频率升高） | 红移（频率降低） | `dopplerFactor(β, cosθ)` |
| 光行差 | 星空向船头聚集 | 船尾方向星空稀疏 | `aberratedCosTheta()` |
| 通讯延迟 | 收到前方信号更早 | 后方地球信号更晚 | 附加 `Δt = distance/c` |

**推荐 UI 设计（既满足用户直觉，又物理正确）：**

```
┌──────────────────────────────────────────────┐
│  驾驶舱仪表板                                 │
│  ┌─────────────┐  ┌─────────────┐            │
│  │ 飞船钟 τ    │  │ 地球钟 t    │  ← 主钟慢对比│
│  │ 12.4 年     │  │ 25.0 年     │            │
│  └─────────────┘  └─────────────┘            │
│  ┌─────────────┐  ┌─────────────┐            │
│  │ 前方观测窗  │  │ 后方观测窗  │  ← 光学对比 │
│  │ 蓝移 λ′/λ   │  │ 红移 λ′/λ   │            │
│  │ 0.42        │  │ 2.38        │            │
│  └─────────────┘  └─────────────┘            │
└──────────────────────────────────────────────┘
```

标签务必写清：**「光学频移」≠「时间速率」**，避免加深误解。

### 5.3 时钟动画实现

```javascript
// 地球系时间推进（已有）
earthTime += dt * timeScale * f(beta);

// 飞船固有时间 — 直接由物理层计算
const { shipTime } = computeRelativityState({ beta, earthTime, earthDistance });

// 仪表盘数字：使用等宽字体，τ 与 t 并排
// 高 β 时让 τ 旁显示 γ 值与「仅过了 τ/γ 的地球年」提示
```

### 5.4 驾驶舱「物理钟」3D 模型

- 船头/船尾各放一个模拟 LED 数字钟模型，**显示相同 τ**（强调共动系同时性）。
- 墙挂「地球直播钟」显示 `earthTime`，旁注「信号延迟 ≈ D/c 年」。
- β > 0.5 时启动「τ 流逝」滴答音效，β 越高音调越低（隐喻时间「变慢」— 注意这是艺术化，非物理必选项）。

### 5.5 静止视角中的双钟

在外部观察场景放置：

- **Earth Clock Tower**：悬浮在地球旁，显示 `earthTime`。
- **Ship Clock Hologram**：跟随飞船，显示 `shipTime`。
- 两条连线标注：「地球已过 25 年 · 船员仅经历 12 年」。

## 6. 尺缩效应：如何模拟与常见误区

### 6.1 两种可视化模式（必须区分）

本项目 `viewMode` 已定义：

| 模式 | 含义 | 尺缩表现 |
|------|------|----------|
| **measured** | 该参考系下的刚性测量结果 | 沿运动方向 scale = 1/γ |
| **observed** | 考虑光学效应后的视觉外观 | 不完全等于 1/γ，需光行差 + Terrell 修正 |

**教学金句**：Measured 回答「尺子量出来多长」；Observed 回答「照相机拍到什么样」。

### 6.2 Measured 模式：正确实现尺缩

**原则**：只在**被观测物体相对于当前参考系存在沿运动方向的速度**时，才对该方向做收缩。

| 观察者参考系 | 被测物体 | 是否收缩 | scale 轴向 |
|--------------|----------|----------|------------|
| 地球系 | 高速运动的飞船 | ✅ | 沿飞船速度方向 |
| 地球系 | 静止的测量杆（固定在场景） | ❌ | 不收缩 |
| 飞船系 | 窗外目标距离标牌 | ✅（距离数值） | 距离 = D/γ，非 mesh 压扁 |
| 飞船系 | 船舱内壁 | ❌ | 静止长度 |

**测量杆示例**（`MeasurementRod.js` 已实现）：

```javascript
const ratio = 1 / lorentzFactor(beta); // lengthContractionRatio
rod.scale.z = ratio;  // measured：Z 轴为杆长方向
```

**飞船外壳**（地球系下）：

```javascript
if (frame === 'earth' && viewMode === 'measured') {
  spacecraft.group.scale.set(
    baseScale, baseScale, baseScale * lengthContractionRatio(beta)
  );
}
```

**注意**：收缩只影响**平行于相对运动方向**的尺寸；垂直于运动方向的截面不变。这是洛伦兹收缩，不是各向同性缩放。

### 6.3 距离收缩 vs 物体尺缩

飞船系中「到比邻星 4.24 ly」变为「1.2 ly」——这是**空间距离在共动系中的测量值变化**，不必把恒星 mesh 压扁。

推荐做法：

- HUD 数字切换 `earthDistance` ↔ `shipDistance`
- 可选：前方距离路标 Sprite 文字更新，而非缩放恒星本身

### 6.4 Observed 模式：超越简单 scale

高速运动时，肉眼图像受以下效应影响：

1. **相对论光行差**：星光向运动方向「聚集」
2. **相对论多普勒**：前方蓝移、后方红移
3. **Penrose-Terrell 旋转**：运动球体看起来被**旋转**而非单纯变扁；β→1 时甚至接近不变形
4. **光传播时间**：看到的是「过去」的发光状态

**简化 Observed 实现**（教学级，本项目 `MeasurementRod` 已用近似）：

```javascript
// observed 模式：杆长略不同于 measured，并附加视觉旋转
rod.scale.z = ratio * 0.92 + 0.08;           // 避免完全压扁
rod.rotation.y = baseRotation + beta * 0.65; // 模拟 Terrell 旋转感
rod.material.color.set(0x88c7ff);              // 与 measured 区分
```

**进阶 Observed 实现**（建议 Phase 2）：

- 自定义 fragment shader，输入 β、视线方向 cosθ，实时计算亮度与颜色
- 参考公式：  
  - 多普勒：`f'/f = γ(1 + β cosθ)`  
  - 光行差：`cosθ' = (cosθ + β) / (1 + β cosθ)`

### 6.5 尺缩交互实验建议

| 实验 | 用户操作 | 预期观察 |
|------|----------|----------|
| 杆与速度平行 | 加速到 0.9c，measured 模式 | 杆明显变短 |
| 杆与速度垂直 | 旋转杆 90° | measured 下长度不变 |
| 切换 observed | 同 β 对比 | 杆外观与 measured 不同 |
| 飞船系 vs 地球系 | 切换 frame | 收缩对象互换（飞船 vs 外界距离） |

### 6.6 常见实现错误清单

| 错误 | 后果 | 修复 |
|------|------|------|
| 全向 `scale.multiplyScalar(1/γ)` | 违反洛伦兹收缩方向性 | 只 scale 运动轴向 |
| 驾驶舱内饰也 scale | 宇航员「变扁」 | 船体共动系内保持 L₀ |
| 把星空粒子距离直接 ×1/γ | 星空几何错乱 | 用光行差改方向，用多普勒改颜色 |
| measured/observed 不区分 | 用户以为尺缩=看见变短 | UI 强制标注模式 |

## 7. 星空光学效应：光行差与多普勒

### 7.1 为什么星空是 VR 沉浸感的关键

近光速下，**窗外星空的变化**是用户最直观的反馈，比 HUD 数字更有冲击力：

- β = 0：各向同性星场
- β = 0.9：星星向船头「聚集」，船尾出现「空洞锥」
- β = 0.99：前方出现强烈蓝白色光晕，后方红暗

### 7.2 光行差（Aberration）

对每颗星方向单位向量 **n**，速度方向 **β**（=cβ 沿运动轴）：

```
cosθ' = (cosθ + β) / (1 + β cosθ)
```

将原方向 **n** 映射为观测方向 **n'**，再重投影到球面。  
本项目 `aberratedCosTheta()` 提供简化标量版；完整实现需对 `StarField` 的 BufferGeometry 做方向变换或 GPU shader。

### 7.3 多普勒（Doppler）

```
f'/f = γ(1 + β cosθ)
```

- cosθ = +1（正前方）：最强蓝移
- cosθ = −1（正后方）：最强红移

颜色映射建议：用 `f'/f` 比值在 shader 中偏移黑体色温或 RGB。

### 7.4 实现路径对比

| 方案 | 性能 | 准确度 | 难度 |
|------|------|--------|------|
| CPU 每帧重算顶点 | 差（8000+ 星） | 中 | 低 |
| Points 自定义 Shader | 优 | 高 | 中 |
| 立方体贴图 + 后处理 | 优 | 中 | 中 |
| 预计算 β 查表 | 最优 | 低 | 低（教学够用） |

**推荐**：Phase 1 用 CPU 批量更新 + 限制星数；Phase 2 迁移到 `ShaderMaterial` 传入 `uniform beta`。

### 8.3 VR 专属设计清单

| 项目 | 建议 |
|------|------|
| 移动 | 左手摇杆：航向；右手摇杆：β 调节；扳机：加速 |

## 附录 A：快速公式卡

```
γ = 1 / √(1 − β²)
Δτ = Δt / γ
L = L₀ / γ        （沿运动方向测量）
D_ship = D_earth / γ
f'/f = γ(1 + β cosθ)
cosθ' = (cosθ + β) / (1 + β cosθ)
```
